Relocation.Audit

The Audit table is used to track changes to the data. When a major change is made in the relocation process, the original data is stored and the changes are stored. If the data needs to be reverted the OriginalDataJSON can be used to revert the data.

Relocation.UhaulInfo

UHaulInfo table stores the ReservationNumber for the UHaul rental agreement. This number is usefull for looking up the rental details on each rental agreement associated with a specific move.

UhaulID
Auto generated ID, Foreign Key in the MoveInfo table.
ReservationNumber
UHaul rental agreement Reservation Number

Relocation.UhaulRental

This table connects the UHaulInfo table with the UHaulRentalTypeReference. One Move may have multiple Rental Agreements (UHaulInfo) and each agreement may contain several different types of rentals (Trailer, Moving Van, Pickup etc.)

Relocation.UhaulRentalTypeReference

Stores the Type of UHaull rental. There may be several types associated with a particular Move. A move may require a 24' Moving Van, a Trailer and a Tow Dolly.

UhaulRentalTypeID
Auto genetated ID
Type
Type of Rental (Trailer, Moving Van, Pickup etc.)
